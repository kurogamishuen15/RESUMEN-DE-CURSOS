MODULO 5 PRINCIPIOS DE COMUNICACIÓN

5.1
PROTOCOLOS DE COMUNICACIÓN:

Para poder brindar informacion utilizamos los protocolos de comunicación, lo cual es la forma de comunicarnos que varia dependiendo del tipo de actividad; Eatá conformado por las siguientes reglas:

  • ¿Cual es método de comunicación que debemos utilizar?
  • ¿Que idiomas debemos usar?
  • ¿Debemos confirmar que se reciben los mensajes?

Metodo: Que tecnicas o pasos debemos seguir para poder comunicarnos (Esto debe indicarse antes de la comunicación

Idioma: Que tipo de lenguaje se va a hablar

Confirmación: Se debe indicar si la comunicación ha sido recibida de forma correcta

Los protocolos son un conjunto de reglas o normas que permiten a los equipos como servidores, computadoras o equipos inteligentes poder intercambiar información, estos definen el tippo de información, la cantidad de datos, que lenguaje se utiliza, gestiona errores, sistema de seguridad etc.

Algunos protocolos que rigen los humanos con exito son:

  • un emisor y un receptor identificados
  • Método de comunicación acordado (Llamada, foto, en persona, etc….)
  • Idioma a utilizar
  • Velocidad o movimiento de entrega
  • Confirmación

POR QUÉ ES IMPORTANTE LOS PROTOCOLOS DE COMUNICACIÓN?

Estos protocolos son importantes ya que permiten el intercambio correcto de información entre los diferentes dispositivos, al ser un conjunto de reglas definidas permite:

  • compatibilidad ya que al tener el mismo idioma los equipos podrán entenderse.
  • Orden: Indican quien habla primero y a que velocidad
  • Integridad de información: Puede activar ciertos protocolos para evitar perdida de información
  • Seguridad: Se indica que tipo se cifrado se utiliza y como se guarda la información para no ser filtrada
  • Encapsulamiento: Cada mensaje debe contener una encabezado que indique el tipo de información, las direcciones, el host etec.
  • formato,tamaño, sincronización, encapsulamiento,partron

DISPOSITIVOS EN UNA BURBUJA

Los dispositivos de red generalmente solo se ven asi mismos como si estuvieran en una burbuja ya que solo saben su información. Para poder comunicarse entre si utilizan los protocolos, ya sea por ejemplo:

  • Protocolos de conexion: son las reglas para conectarse ya sea de forma inalambrica o alambrica
  • DNS: Sirve para buscar los dominios y poder entablar una comuniación
  • IP: Es el identificador de los dispositivos y permite que la información llegue a la dirección solicitada

INTERNET Y LOS ESTÁNDARES

Tras la creciente de internet y los dispositivos se ha inventado nuevas formas de comunicación es decir los estandares

Son conjuntos de reglas que determinan los pasos y el orden en la comunicación, esto ayuda a que los dispositivos sean compatibles entre si y puedan hacer uso de toadas las caracteristicas. Estos estandares permiten la comunicación en internet

ORGANIZACIONES DE ESTANDARES

Un estándar de Internet es el resultado final de un ciclo completo de discusión, resolución de problemas y pruebas. Estos distintos estándares son desarrollados, publicados y mantenidos por diferentes organizaciones, tal como se indica en la figura. Cuando se propone un nuevo estándar, cada etapa del desarrollo y del proceso de aprobación es registrada en un documento numerado de solicitud de comentarios (RFC, Request for Comments) para seguir la evolución del estándar. Las RFC para los estándares de Internet son publicadas y administradas por el Grupo de Trabajo de Ingeniería de Internet (Internet Engineering Task Force - IETF).

5.4

PROTOCOLOS DE RED:

En internet utilizamos distintos protocolos de comunicación

Ethernet: Es el protocolo de comunicación fisico que permite conectar dispositivos mediante un cable de red

IP: Es el protocolo: de identificadores de los disposirivos mediante una dirección

TCP: Es un protocolo de verificación de paquetes, este se encarga de la negociación para evitar la perdida de información

Http: Es un protocolo web para la apertura del lenguaje de hipertexto HTML

PILA DE PROTOCOLOS / TCP:

La pila de protocolos es una serie de pasos o jerarquia donde cada capa tiene una funcion especifica para la comunicación, en este caso utilizaremos el TCP/IP

NETWORK ACCESS (ACCESO DE RED): Es la comunicación entre nuestra red interna, permite que tengan un elemento físico por el cual pasar la información en este caso es el ETHERNET

INTERNET: Para poder comunicarnos entre nuestras redes o otras redes se necesita un identificador uníco en el cual encontrar nuestro dispositivo para eso se utilizan las IP que pueden ser de version IPV4 o IPV6

TRANSPORT: Se encarga de enviar los archivos, determinar quien habla primero, verifica los paquetes, negocia la comunicación, verifica el contenido, etc.. Para eso se usa el TCP

APLICATION (ES LA APLICACIÓN O EL FIN DE USO): Permite visualizar o utilizar ese contenido, el mas usado es el HTTPS que se usa para las webs

MODELO DE REFERENCIA OSI:

Tenemos 2 tipos de modelos para describir la información que deben estar presentes:

Protocolo: Los los conjuntos de protocolo en particular requeridos para el intercambio de información, indica claramente que se debe realizar para que las personas se comuniquen con una red de datos (TCP/IP

Referencia: Este modelo indica las funciones que se deben realizar para la comunicación ,pero no detalla específicamente como se debe realizar esa función y no define de forma precisa lal manera en la cual debe funcionar el protocolo

COMPARACIÓN ENTRE OSI Y TCP

Debido a que TCP/IP es el conjunto de protocolos en uso para las comunicaciones por Internet, ¿por qué también necesitamos aprender el modelo OSI?

El modelo TCP/IP es un método para visualizar las interacciones de los diversos protocolos que conforman el conjunto de protocolos TCP/IP. No describe las funciones generales necesarias para todas las comunicaciones de red. Describe las funciones de red específicas de los protocolos en uso en el conjunto de protocolos TCP/IP. Por ejemplo, en la capa de acceso a la red, la suite de protocolos TCP/IP no especifica los protocolos que se deben utilizar para transmitir a través de un medio físico, ni el método de codificación de las señales para la transmisión. Las capas 1 y 2 de OSI tratan los procedimientos necesarios para acceder a los medios y las maneras físicas de enviar datos por la red.

Los protocolos que forman la suite de protocolos TCP/IP pueden describirse en términos del modelo de referencia OSI. Las funciones que se producen en la capa de Internet del modelo TCP/IP se incluyen en la capa de red del modelo OSI, tal como se indica en la figura. La funcionalidad de la capa de transporte es la misma en ambos modelos. Sin embargo, la capa de acceso a la red y la capa de aplicaciones del modelo TCP/IP se dividen a su vez en el modelo OSI para describir funciones discretas que deben realizarse en estas capas.
